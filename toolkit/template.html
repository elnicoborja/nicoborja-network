LECTURA 02 · COMPOSICIÓN DE ROLES

Quién contesta primero el teléfono.

La red se concentra en decisores y operadores senior, no en juniors. Eso cambia el método: no escala por volumen, escala por multiplicador. Tamaño del bloque = cantidad de conexiones que cargan ese rol en su título.

CÓMO SE HIZO

El método.

  1. 01Export oficial de LinkedIn (Settings > Data Privacy > Get a copy of your data). Llega un ZIP con Connections.csv, Skills.csv, Endorsements.csv.
  2. 02Parser local (Python) que normaliza, deduplica, infiere seniority desde el headline, y agrega era profesional según fecha de conexión.
  3. 03Scoring 0-100 sobre cuatro señales declaradas: seniority, ICP fit (industria + keywords), pista LATAM, warmth (endosos + reinvitaciones).
  4. 04Output: un JSON con las ocho lecturas de arriba, inyectado en este HTML. PawBot también lo consume para el digest matinal.

QUIERES HACER LO MISMO CON TU RED

El toolkit.

Las ocho lecturas se generan a partir de un ZIP de LinkedIn. Tomé el método y lo empaqueté como un parser Python + un template HTML + un SKILL.md para que cualquier LLM lo aplique a tu propia exportación. Es portable, local, sin enviar tu data a ningún sitio.

QUÉ TRAE EL TOOLKIT
  • →SKILL.md: instrucciones para Claude / Cursor / cualquier LLM agéntico. Pegas el ZIP, te devuelve tu HTML.
  • →parser.py: script Python que parsea Connections.csv y produce el JSON de las ocho lecturas.
  • →template.html: este mismo archivo, con un slot vacío donde el parser inyecta tus datos.
  • →README.md: pasos para exportar de LinkedIn, correr el parser, y publicar tu versión.
PARA QUIÉN ES
  • →Operadores que llevan más de 5 años en LinkedIn y no saben qué hay dentro de su red.
  • →Founders que necesitan una capa de inteligencia sobre sus contactos sin pagar Apollo o Clay.
  • →Cualquiera buscando cambiar el método: del broadcast al uno a uno.
  • →Tú no necesitas saber Python. Necesitas un LLM que lea SKILL.md.
